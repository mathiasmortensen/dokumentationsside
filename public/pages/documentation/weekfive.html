Serving static files in Express

When you set up a web application you want to limit how much the users of site can have access too.

In some way a web application allows users to access data on a remote server, and that is why access has to be limited.

Assume all the neccessary lines of code are already written e.g (app.get('/', (req, res) => {res.send({data: "it works!"})})); 


app.use(express.static('assets'));

We use this chunk of code to tell the application that users can be served files that are in the 'assets' directory.

it is also vital that when you start the application you need to be in the directory that contains the 'assets' directory, otherwise this probably won't work.

Exporting and importing in frontend (type="module" attribute)

Modules is one way to make a JavaScript file have access to functions or other neccessary chunks of code in another JavaScript file.

The first thing that needs to be done is editing the package.json in your project. 


{
"dependencies": {
"express": "^5.2.1"
}
}

This is just your plain simple package.json, however this does not make it possible to export or import modules.. For this to work you need this... 


{
  "type": "module",
  "dependencies": {
    "express": "^5.2.1"
  }
}

With this code, it allowes you to import "modules" that are exported from other files.. 


FILE: app.js
import express from 'express';

const app = express();

import pagesRouter from './routers/pagesRouter.js';

FILE: pagesRouter.js
import { Router } from 'express';

import { weekOnePage, weekTwoPage, weekThreePage, weekFourPage, weekFivePage, weekSixPage, indexPage } from '../util/pagesUtil.js';

const router = Router();

router.get('/', (req, res) => {
res.send(indexPage);
});

// MORE CODE

export default router;

When the module is exported at the bottom, it is now accessible to other JavaScript files, if imported.

Commonjs in JavaScript

Commonjs is another module system that is a bit outdated but can still be used.

The syntax is different when 'importing'.

Instead of using the 'import' statement you use the keyword require. Example below: 


const express = require('express');

Commonjs vs ESM

Use commonjs when mainting old legacy-code and use ESM (module) type for new and modern development.

Redirection in Express

At first i will go over the built in functions in Express.. (SERVER SIDE REDIRECTION) 


res.redirect('/new-site');
res.redirect(301, 'https://movedpermanently.com');

You can use this code in a route, and then it will redirect the user based on the url.

With the next two methods you will be able to do a client-side redirection, because the user is sent the site, and then the redirection happens.

However you can also use the DOM that is provided with native JS.


window.location.href = '/new-site';

This will redirect the user as well.

The last way (that i know of), you can use to redirect the user is pure HTML. 


<meta http-equiv="refresh" content="0; url=/new-page">

This will redirect the user when accessed.
