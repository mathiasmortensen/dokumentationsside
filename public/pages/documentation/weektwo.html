HTML

HTML stands for HyperText Markup Language, and it is basically the standard language for creating web pages.


<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
<h1>This is a simple HTML header</h1>

<p>This is a simple HTML paragraph.</p>
</body>
</html>

It parses the HTML tags and renders the content in the browser.

Express - A Web Framework for Node.js

Express is a framework that allows developers to build web applications and API's with ease.

It provides a bunch of features that make it easy to create a web application.

So how does it work?


const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('Hello, World!');
});

const port = 8080;

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});

In the code above, firstly Express is imported with `require('express')` and an instance called 'Express' is created.

After this, we instansiate the variable `App` with the Express instance.

With this, we can now define routes on our app instance.

In this case, we define a GET route on the root path ('/') that sends back 'Hello, World!' in HTML format.

At last a port is defined on the variable `port, and it is then used in the `app.listen(port, () => {console.log(`Server running at http://localhost:${port}`)});`

This makes the server start on http://localhost:8080

Build Tools

Build tools are used to automate tasks in the development process, such as bundling, running tests, and starting the application.

The one we've been using is Npm (Node Package Manager).

NPM uses a package.json file to determine the dependencies and scripts for a given project.


{
  "type": "module",
  "dependencies": {
    "express": "^5.2.1"
  }
}

In the code above, we have a package.json file that specifies that our project uses a module type and has a dependency on the Express framework

The package.json file can also be used to define custom scripts for the project environment.


{
  "type": "module",
  "dependencies": {
    "express": "^5.2.1"
  },
  "scripts": {
    "start": "node index.js"
  }
}

This script that we have defined allows us to start the application by running `npm run start` or `npm start`

XSS (Cross-Site Scripting)

XSS is a type of security vulnerability that allows hackers to inject malicious scripts into web pages.

This particular vulnerability can be exploited and in the end it can lead to remote code execution.


// EXAMPLE OF XSS VULNERABILITY

const containerDiv = document.getElementById('container');

<img src="http://google.com/img.png" onerror="alert('XSS Vulnerability!')">

In the code above, we have an image tag with an `onerror` attribute that executes a JavaScript alert when the image fails to load. This is a simple example of an XSS vulnerability. This allows the attacker to execute JavaScript code on the server.
