CRUDable REST API - Part II

On this page, i will show how to implement this in express. This is mainly because the topic on the last page was more theoretic


        const express = require('express');
const app = express();

app.use(express.json());

let movies = [
    { id: 1, title: 'Garlic is as good as ten mothers', year: 1976 },
    { id: 2, title: 'Microscopic Liquid Subway to Oblivion' }
];

let nextId = 3;

app.get("/movies", (req, res) => {
    res.send({ data: movies });
});

app.get("/movies/:id", (req, res) => {
    const providedMovieId = Number(req.params.id);
    const foundMovie = movies.find((movie) => movie.id === providedMovieId);

    if (!foundMovie) {
        return res.status(404).send({ errorMessage: `No movie found by id: ${req.params.id}` });
    }
    res.send(foundMovie);
    
});

app.post("/movies", (req, res) => {
    if (!req.body) {
        return res.status(400).send({ errorMessage: 'No JSON body provided.' });
    }

    const providedMovie = req.body;

    providedMovie.id = nextId++;

    movies.push(providedMovie);

    res.send({ data: providedMovie });
});

app.patch("/movies/:id", (req, res) => {
    const providedMovieId = Number(req.params.id);
    const foundMovieIndex = movies.findIndex((movie) => movie.id === providedMovieId);

    if (foundMovieIndex === -1) {
        res.status(404).send({ errorMessage: `No movie found by id: ${req.params.id}` });
    }

    const foundMovie = movies[foundMovieIndex];
    const providedMovie = req.body;

    const movieToCreate = { ...foundMovie, ...providedMovie, id: providedMovieId };
    movies[foundMovieIndex] = movieToCreate;

    res.send({ data: movieToCreate });
});


app.delete("/movies/:id", (req, res) => {
    const providedMovieId = Number(req.params.id);
    const foundMovieIndex = movies.findIndex((movie) => movie.id === providedMovieId);

    if (foundMovieIndex === -1) {
        return res.status(404).send({ errorMessage: `No movie found by id: ${req.params.id}` });
    }

    movies.splice(foundMovieIndex, 1);

    res.status(204).send();
 });

app.listen(8080);

In the example above a simple functioning CRUD API is demonstrated.

CRUD stands for Create, Read, Update and Delete, which are the four basic operations that can be performed on a resource.

In this exact example we have implemented all four operations for managing a collection of movies.

We have 5 routes tho, which is because of the two get methods. One is for getting all the movies, while the other is for a specific movie.

Fetch

Fetch is a built-in function in JavaScript and it it used to make HTTP requests from the browser.


fetch('http://mathiasmortensen.dk/vigtigjson')
  .then(response => response.json())
    .then(data => console.log(data))
    .catch(error => console.error('Error:', error));

In the code above, we are making a GET request to the URL http://mathiasmortensen.dk/vigtigjson and then we are logging the response data to the console.

Fetch returns a promise, which is why we can use .then() to handle the response.

Time - using Date

The Date object in JavaScript is used to work with dates and times.

Date.now() returns a timestamp in the Unix Epoch Time format (which is milliseconds, since Jan. 1st 1970)


            const timestamp = Date.now();
            console.log(timestamp); // milliseconds since Jan. 1st 1970

            const day = new Date().getDay();
            console.log(day); // returns the day of the week (0-6)

            const weekDay = new Date().toLocaleString('en-dk', { weekday: 'long' });
            console.log(weekDay); // returns the day of the week in long format (e.g. "Monday")

            const month = new Date().getMonth();
            console.log(month); // returns the month (0-11)

            const year = new Date().getFullYear();
            console.log(year); // returns the year (e.g. 2026)

Deployment with dedicated server (home brew version)

We got taught a way to deploy with a website called vercel

The method we used to deploy was interchangable, and i have a standby server at home, which is why I didn't use vercel...

This will be an almost complete and thorough guide to deployment on a dedicated server.. HEADLESS

Server Setup

First we need an operating system on the dedicated server. I use debian for reference.

When the headless server is done setting up, then the first step is complete.

Network Configuration (Skip if using dedicated server, unless you have problems accessing the deployed application)

To access the application on the web, the port that your application is using needs to be forwarded or open

You need to open the port on the operating system, if you use UFW, the command is simply

ufw allow 8080 This will open the port both on UDP and TCP. TCP is recommended so you minimize packet loss.

If this is done on a machine at home, you will likely need a static ip address.

If you have a static ip address the process is rather simple now.

You will need to access the interface which you can access on the browser, using the default gateway (often its just, http://192.168.1.1)

To find the default gateway you need net-tools. Install it by using this chunk of text: sudo apt get install net-tools

this will make the ifconfig command work.

Runtime and neccessary tools!

This course uses Node.js, so you need to install nodejs on the server, npm, git and screens.

Once these programs are installed you're almost set to go!

If your repository is public you do not need to login when using the git commands..

Next step is cloning your repository onto your server using the command git clone https://www.github.com/exampleUser/exampleProject

Once the cloning process is done, you want to change directory into the folder containing your repository code..

Then you install all of the dependencies, using the npm install command.

Now you want to create a screen session, so when you close out of your ssh-session the application wont exit..

Type screen in the terminal, after this you press Ctrl + A followed by c.

Now you're inside of a screen session!

Now comes the fun part, you simply start your application! run node app.js and the application should start and be available!
