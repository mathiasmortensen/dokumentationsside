Scripts with NPM

In the package.json included with projects that utilize NPM

Lets say you want a custom script that installs the dependencies of a project and run the application right after. In this case you would do this: 


{
  "name": "test",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "installnrun": "npm install && node app.js"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "type": "commonjs"
}

Here a script called installnrun is created.. When you run this command in the terminal, it will then execute the script: npm run installnrun

Native Environment Variables in Node.js

You can define the port with the variable inside of process.env.PORT

It is also possible to set temproray values to variables with the export PORT=8080 command inside your shell.

Node.js also reads the port exported from the terminal.

Semantic HTML

This topic is all about giving your code meaning and not only looks.

Instead of just using divs for everything, you use tags that tells Search Engines and the browser what the content of site actually is. 


<main>
    <h1>Headline</h1>
    <section>
        Content HERE
    </section>
    <footer>Bottom Text</footer>
</main>

How to make a footer sticky!

You really dont want a footer floating arround on your page, which is why there are a couple solutions to this.

Let us presume your HTML looks like this:


<body>
  </Indhold</main>
  </footerBund</footer>
</body>

To make the footer always stay at the bottom of the page as in not being sticky (glued to your view..), then you would need some CSS.. 


body {
    display: flex;
    flex-direction: column;
    min-height: 100vh;
}

main {
    flex: 1;
}

With this; you body is forced to be filled with all space until it reaches the footer.

What if you always want to be able to see the footer?

The same HTML will be used for this example:


footer {
    position: sticky;
    bottom: 0;
}

Which is better?

It all depends on what you prefer and if you can promise to serve x files.

If you dont have enough space on the server you deploy on, using CDN can be nice, since you dont need to have it locally then.

However if you know people from china are going to visit your web application, and you deploy on a chinese server in China, then you probably shouldn't serve files from servers in europe.
