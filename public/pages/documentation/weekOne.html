Introduction to JavaScript

This document covers most of the basics of JavaScript, and it is suposed to represent the first class of the course.

Variables

In JavaScript there are three ways to declare variables:

  • const
  • let
  • var

One of the most basic and important rules for Javascript, is to choose the right variable declaration type.

  • Use const if the value will not change
  • Use let if the value must change
  • Avoid var. This is pretty much outdated.

const

A const variable cannot be reassigned, but objects stored in it can still change internally. What i mean by this is that you can change the properties of an object declared with const, but you cannot reassign the variable to a new object. 


const user = { name: "Mathias" };

user.name = "Victor"; // allowed

// user = {} // error

Why var is problematic declaration type:

var has function scope and is hoisted. This can cause unexpected behavior. 


console.log(a);

var a = 5;

Because of hoisting the variable exists before it is declared, but its value is undefined in this context.

Scope

The scope determines where a variable is accessible in JavaScript.

  • Global scope
  • Function scope
  • Block scope

Block scope

Variables declared with let and const are block scoped. 


if (true) {
  const message = "Inside block";
  console.log(message);
}

console.log(message); // error - outside of block

  • String
  • Number
  • Boolean
  • BigInt

  • undefined

const name = "Mathias";
const age = 30;

const person = { name: "Mathias" };

const empty = null;
let notDefined;

null represents a value that is inherently empty. while undefined represents a variable that is not yet assigned.

Type Coercion

Type coercion occurs when JavaScript automatically converts one data type into another. 


const number = 17;
const stringNumber = "17";

console.log(number + stringNumber);

This gives the output "1717" because the number is converted to a string.

Equality Operators


5 == "5"   // true
5 === "5"  // false
  • == performs type coercion - (5 is equal to 5, hence true)
  • === compares value and datatype - (5 is not strictly equal to "5" because the variable that holds the reference is not the same.)


const k = 5;
const l = "5";

if(k === l) {
  console.log("Equal");
} else {
  console.log("Not equal");
}

if(k == l) {
  console.log("Equal");
} else {
  console.log("Not equal");
}

ASI (Automatic Semicolon Insertion)

JavaScript can automatically insert semicolons at the end of lines.

However, ASI can cause unexpected bugs. 


function getUser() {
  return
  {
    name: "Mathias"
  }
}

console.log(getUser());

Because a semicolon is inserted after the return, the function returns undefined.

"use strict"

Strict mode enables stricter parsing and error handling in JavaScript. It almost acts as a linting tool built into the language, and helps catch common coding mistakes. 


"use strict";

x = 10; // ReferenceError

Without strict mode this would create a global variable accidentally.

Functions

Functions are reusable lines of code.

Function Declaration


function greet(name) {
  return "Hello " + name;
}

Anonymous Function


const greet = function(name) {
  return "Hello " + name;
};

Arrow Function


const greet = (name) => `Hello ${name}`;

Functions as First-Class Citizens

Functions can be passed as arguments. 


function processUser(callback) {
  callback("Mathias");
}

processUser((name) => console.log(name));

Ternary Operator

The ternary operator is an alternative to an if/else statement. 


const age = 20;

const result = age >= 18 ? "Adult" : "Minor";

Spread Operator

The spread operator ... expands the elements of an array or object. 


const user = { name: "Mathias", age: 30 };

const updatedUser = { ...user, age: 31 };

JSON

JSON is a data format used for storing and exchanging data. 


{
  "name": "Mathias",
  "age": 30,
  "active": true
}

Keys are always declared as string in JSON.

REST API Design

REST (Representational State Transfer) is an architectural style for designing networked applications.

One of the many ideas behind REST is to make web applications simple, scalable, and maintainable.

Resource identification is a key concept in REST, where each resource is identified by a unique URI (Uniform Resource Identifier).

It is also important to note that REST APIs are stateless, meaning that each request from a client to a server must contain all the information needed to understand and process the request.

Method Operation
GET Retrieve data
POST Create resource
PUT Replace resource
PATCH Partial update
DELETE Remove resource

Example API


GET /items
GET /items/{id}
POST /items
PUT /items/{id}
PATCH /items/{id}
DELETE /items/{id}
