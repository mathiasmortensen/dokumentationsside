What is Nodemon?

Nodemon is a tool that helps developers to debug applications while they are being developed.

One of its core features is auto-restarting the node application when file changes in the directory are detected.

Another one of its features is providing real-time feedback during development. 


npm install -g nodemon


nodemon nameOfYourFile.js

With this chunk of text, you install nodemon globally on your computer hence the -g flag, when it is done installing, you can use nodemon simply by typing "nodemon application.js" and it will be started.

Loops and loop methods


for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log("var:", i), 100);
}

for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log("let:", i), 100);
}

Because var does not create block scope, all callbacks reference the same variable. Here you can truly see as to why var is problematic.

Loop Methods

Modern JavaScript prefers loop methods over traditional loops.

forEach


const numbers = [1,2,3];

numbers.forEach(n => console.log(n));

map


const prices = [100,200,300];

const taxed = prices.map(p => p * 1.25);

filter


const ages = [12,18,25];

const adults = ages.filter(age => age >= 18);

reduce


const numbers = [10,20,30];

const total = numbers.reduce((sum,n) => sum + n, 0);

find


const users = [
  {name:"Mathias", active:false},
  {name:"Victor", active:true}
];

const activeUser = users.find(u => u.active);

Why Loop methods are really good

  • Readability
  • Declarative
  • Less boilerplate
  • More predictable

Anatomy of an URL

However, understanding the structure of a URL is often overlooked and tossed aside because of how easy it is to use without understanding.

endpoint /search
query string / query parameters ?hl=en&q=anatomy%20of%20an%20url

Parameters in requests

As mentioned in the previous "chapter", we have something called query string and query parameters (they are the same thing)

When making a post request, we can also include parameters in the request body.

We also have something called path parameters, which are parameters that are included in the URL path.

For example, in the URL https://example.com/users/123, the 123 is a path parameter that can be used to identify a specific user.

While query parameters are used to pass data in the URL query string, path parameters are used to identify resources in the URL path.
